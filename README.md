# YouTube Pure Player（极简 YouTube 播放器）

一个桌面应用：打开后就是 YouTube，**观看视频时只显示画面本身，看不到任何按钮和文字**。
支持登录你的 YouTube 账号，内置广告屏蔽。

## 功能

- **纯净观看**：进入视频页自动隐藏顶栏、侧栏、标题、简介、评论、推荐、播放器控件等一切元素，只剩视频画面。
- **可登录**：和普通浏览器一样登录 Google / YouTube，登录状态会本地保存，下次打开仍是登录状态。
- **内置广告屏蔽**：在网络层拦截广告与追踪请求（基于 Ghostery 过滤列表），跳过/屏蔽大部分视频广告。
- **选择音频设备**：在 `Ctrl+L` 面板中可自由选择输出设备（声音从哪个扬声器/耳机播放）和输入设备（麦克风），选择会被记住并自动应用到视频。
- **多语言**：在 `Ctrl+L` 面板中选择「语言」（中文 / English / Русский / Français / Español / Українська / ქართული / Հայերեն / Malagasy），**一次切换同时改变软件界面文字和 YouTube 页面显示语言**，选择会被记住。
- **隐藏式窗口控制**：右上角常驻 设置(⚙) / 最小化 / 最大化还原 / 关闭 四个按钮，平时透明且不挡点击，**鼠标移到右上角才整体淡入显示**，不影响观看。点击 ⚙ 即可打开/关闭设置面板（等同 `Ctrl+L`）。
- **无标题栏（适合屏幕共享）**：窗口没有标题栏，共享窗口给 FB / Zoom / 其他通话时，**不会暴露 “YouTube” 文字和图标**；任务栏标题也固定为中性名称。
  - 移动窗口：拖动顶部中间的透明区域；双击该区域可最大化。
- **干净无边框**：没有系统菜单栏、没有多余 UI。

## 快捷键

| 按键 | 作用 |
| --- | --- |
| `Ctrl + L` | 开关设置面板（同一个键开/关）；面板内可输入链接、选语言、选音频设备。点击面板外的空白处或按 `Esc` 也可关闭 |
| `Ctrl + H` | 切换纯净模式（纯净观看 ⇆ 浏览/搜索/登录） |
| `Esc` | 退出纯净模式 |
| `F11` | 全屏 / 退出全屏 |
| `Ctrl + R` | 刷新 |
| `Ctrl + M` | 最小化 |
| `Ctrl + Q` / `Alt + F4` | 关闭窗口 |
| `Alt + ←` / `Alt + →` | 后退 / 前进 |
| 空格 / `K` | 播放 / 暂停 |
| `←` `→` / `J` `L` | 快退 / 快进 |
| `↑` `↓` | 音量 |
| `F` | YouTube 原生全屏 |

> 纯净模式下没有可见的控制条，全部用键盘或点击画面（点击=暂停/播放）操作。
> 需要搜索、选视频或登录时，按 `Ctrl + H` 切回浏览模式即可。

## 使用方式

1. 安装依赖（首次）：

```bash
npm install
```

2. 启动：

```bash
npm start
```

### 典型流程

1. 启动后按 `Ctrl + H` 切到浏览模式，登录账号、搜索或挑选视频。
2. 点开任意视频 —— 会**自动进入纯净模式**，只剩画面。
3. 想换视频时按 `Ctrl + H` 回到浏览模式，再选下一个。

## 打包成 exe（可选）

```bash
npm install --save-dev electron-builder
npm run dist
```

生成的安装包在 `dist/` 目录下。

## 说明

- 广告过滤列表首次启动时联网下载并缓存到本地，之后离线也可用。
- 登录信息保存在 Electron 的用户数据目录（`persist:youtube` 分区），仅存于本机。
- 本工具仅是 YouTube 网页的本地外壳，需遵守 YouTube 使用条款。

## 如何发布新版本

本项目使用 GitHub Actions 自动构建和发布。每次发布新版本只需要创建一个 Git Tag 并推送即可。

### 发布步骤

#### 1. 确保代码已提交并推送

在发布之前，确保你的所有代码改动已经提交并推送到 GitHub：

```bash
# 查看当前状态
git status

# 添加所有改动
git add .

# 提交改动（把"你的改动说明"替换成实际的描述）
git commit -m "你的改动说明"

# 推送到 GitHub
git push origin main
```

#### 2. 创建版本 Tag

Git Tag 是一个版本标记，用于标识发布的版本号。版本号格式为 `v主版本.次版本.修订版本`，例如 `v1.0.0`、`v1.1.0`、`v2.0.0`。

```bash
# 创建一个新的版本 tag（将 v1.0.1 替换为你想要的版本号）
git tag -a v1.0.1 -m "Release version 1.0.1"
```

#### 3. 推送 Tag 触发自动构建

```bash
# 推送 tag 到 GitHub（这会自动触发 CI 构建）
git push origin v1.0.1
```

推送后，GitHub Actions 会自动执行以下操作：

1. 构建项目（在 Windows 上用 electron-builder 打出安装包）
2. 生成安全签名（Attestation）
3. 创建 Release 并上传构建产物（`.exe` 安装包等）

#### 4. 查看构建结果

- 构建进度：访问项目的 **Actions** 页面查看
- 发布结果：访问项目的 **Releases** 页面查看已发布的文件

### 版本号说明

| 版本号格式 | 什么时候用 | 示例 |
|-----------|-----------|------|
| `vX.0.0` | 重大更新、不兼容改动 | `v2.0.0` |
| `vX.Y.0` | 新增功能 | `v1.1.0` |
| `vX.Y.Z` | 修复 bug | `v1.0.1` |

### 如果构建失败怎么办

1. 访问项目的 **Actions** 页面查看错误日志
2. 修复代码问题
3. 删除失败的 tag 并重新创建：

```bash
# 删除本地 tag
git tag -d v1.0.1

# 删除远程 tag
git push origin :refs/tags/v1.0.1

# 修复问题后，重新创建并推送
git tag -a v1.0.1 -m "Release version 1.0.1"
git push origin v1.0.1
```
